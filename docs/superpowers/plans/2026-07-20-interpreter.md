### Task 2: `shared/event-interpreter.ts` — the pure stepper

`RunContext` + `stepEventRun(context, program, state, rng?) → { context, effects[] }` executing exactly ONE command; effects are data (`{ kind: "say", ... } | mutateState | teleport | gold | items | closeDialogue`). Control flow: `if` resolves against the state snapshot and splices the taken branch into the pc path (or a frame stack — implementer's choice, justified); `loop`/`breakLoop` via the loop stack; `wait` returns `waiting-timer` with a tick deadline the CALLER computes (the stepper stays clockless); `say`/`choices` return `waiting-advance`/`waiting-choice`; `choose` input resumes with the chosen branch... (choices in the wireframe are value-only — the RESULT is exposed as the next `if` reading a designated variable? NO — simplest honest model: `choices` carries per-option `body[]` blocks, XP-style. Decide and document). `exitRun` → done. Tests: per-opcode table; programs (nested if in loop with break; choices branch execution); the budget property — a `while(true)`-equivalent loop stepped N times yields N steps and never diverges; determinism (same inputs → same effects). Mutation proofs: branch-splice inversion (else taken when condition holds → table fails); loop-stack underflow guard.

### Task 3: runtime — `event-run-system.ts` + the real coordinator writes

`World`: trigger detection (interact intent near an `action` event → start; movement onto a `player-touch` event cell → start; both only for `normal`-kind events with a satisfied active page); the room-local run lock (Q4 — second trigger dropped while a context lives); the per-tick drain (≤ `EVENT_COMMANDS_PER_TICK` across all contexts, round-robin); effect dispatch (say/choices → the wire to the TRIGGERER only; mutateState → coordinator RPC; teleport same-map = authoritative position set / cross-map = the existing exit-handoff path with the event's destination; gold/items → the hero's session runtime); context abort on disconnect/transition/death; distance-close (each drain tick, if a context waits on dialogue and the triggerer is beyond `DIALOGUE_CLOSE_RADIUS`, emit closeDialogue and abort the wait — decide abort-vs-resume-at-pc and document; WoW closes the panel, the run ENDS). `GameSession`: `#applyStateChange` goes real — mutations carry the run's identity, apply serially, bump a **monotone `version`** shipped with every snapshot; `installAdventureState` gains the `>=` guard (never-throw preserved); the debounce becomes **`ctx.storage.setAlarm`**. Tests against the real DO harness: trigger→run→flip→cross-room visibility (extends the t4 suite); double-trigger lock; budget starvation (an authored infinite loop: room keeps ticking, movement stays live — assert another hero still moves); out-of-order install dropped by the version guard; abort on disconnect. Mutation proofs: lock removed; guard removed; budget unbounded (the starvation test hangs → fail via timeout ceiling, document).
