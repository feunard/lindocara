# @lindocara/ui

The stock **shadcn** component tree (base-nova / neutral) — the vocabulary for creator tools and any
non-game surface. Depends only on npm packages (no engine/renderer). This is one of the two component
trees: **this one for creator tools, `client/ui/tiny-swords/` for player/game UI.**

## Responsibility

- `src/components/*.tsx` — the shadcn primitives (button, input, dialog, label, tooltip, menubar,
  dropdown-menu, toggle, toggle-group, resizable). Generated by `shadcn add` — do not hand-edit.
- `src/lib/utils.ts` — `cn()`. `src/styles/globals.css` — the shadcn base tokens (`@theme`,
  `:root`/`.dark`, `@layer base`); the client's `app.css` imports it **last** so these win over the
  Tiny Swords sheets. `components.json` — this package's shadcn config (monorepo mode).

## Graph

- **Depends on:** npm only (`@base-ui/react`, `class-variance-authority`, `clsx`, `lucide-react`,
  `react`, `react-dom`, `react-resizable-panels`, `tailwind-merge`).
- **Depended on by:** `client`, `editor` (import `@lindocara/ui/components/<name>.js`).

## Commands

```bash
npm run typecheck:ui        # tsc (no tests: pure component vocabulary)
npm run ui:add -- <name>    # = shadcn add -c packages/ui  (monorepo mode), then npm run lint:fix
```

## Rules

- Never hand-edit `src/components/`: `shadcn add` regenerates it against the upstream token set.
- Never import a Tiny Swords component here, and never import a ui component into a game surface —
  the two-tree rule. Components read shadcn tokens; Tiny Swords reads its own `--tiny-*` tokens.
- Add a component with `npm run ui:add` (monorepo mode resolves aliases from this `components.json`),
  then `npm run lint:fix` (stock shadcn output has no semicolons; Biome requires them).

See the root [`AGENTS.md`](../../AGENTS.md) for the two-tree rule and the `legacy.css` fence.
